Beginner Level · Weeks 1–8
The Foundations
Start here. No prior experience needed. Master these before moving on.
✅
Must-Learn-First Priority: AI → Machine Learning → Deep Learning → Neural Networks → NLP → LLMs. This is the exact order. Don't skip steps.
🤖 What is AI?
3–4 days
Simply: AI means teaching computers to do tasks that normally need human intelligence — like reading, writing, seeing, and making decisions. Your phone's face unlock is AI. Netflix recommendations are AI.
Why it mattersAI is the entire field you're entering. Understanding it at a high level gives you context for everything else.
Real exampleGmail's spam filter automatically learns which emails are junk — that's AI.
🔨 Mini ProjectTry 5 different AI tools in one day: ChatGPT, Google Lens, Spotify recommendations, Siri/Alexa, and Grammarly. Write what you notice.
ResourcesGoogle's "AI For Everyone" (free), 3Blue1Brown YouTube, AI explained by IBM
📊 Machine Learning
1 week
Simply: ML is a type of AI where computers learn from examples instead of being programmed with exact rules. You show the computer thousands of cat photos, and it learns what cats look like on its own.
Why it mattersLLMs are a type of ML model. You need to understand how learning from data works.
Real exampleAmazon product recommendations — ML looks at what you and similar users bought, then predicts what you'll like.
🔨 Mini ProjectUse Google's Teachable Machine (free, no code) to train a model that recognizes your hand gestures with your webcam.
ResourcesAndrew Ng's ML course on Coursera (audit free), StatQuest YouTube, fast.ai
🧠 Deep Learning
1 week
Simply: Deep Learning is a powerful type of ML that uses multiple layers of math (like a brain with many layers of neurons). It's what makes image recognition, speech recognition, and LLMs work.
Why it mattersEvery modern AI system — ChatGPT, DALL-E, Stable Diffusion — is built on Deep Learning.
Real exampleGoogle Photos automatically groups your photos by person — deep learning recognizes faces.
🔨 Mini ProjectRun a pre-trained image classifier in Google Colab (free). Feed it photos and see what it predicts. No setup needed.
Resourcesfast.ai Practical Deep Learning (free), Deep Learning Specialization - Coursera
🔗 Neural Networks
5 days
Simply: A neural network is a computer system loosely inspired by the human brain. It has layers of nodes (like neurons) connected together. Data flows through the layers, and the network learns which connections matter most for making accurate predictions.
Why it mattersLLMs are giant neural networks. Understanding the basic structure helps everything click.
Real exampleSiri understanding your voice — a neural network converts sound waves into text, then understands the meaning.
🔨 Mini ProjectUse Playground.tensorflow.org (browser-based, no install) to visually train a tiny neural network and see how it learns.
Resources3Blue1Brown Neural Networks series (YouTube), Michael Nielsen's free online book
💬 NLP (Natural Language Processing)
1 week
Simply: NLP is about teaching computers to understand human language — reading, writing, and speaking. It handles tasks like translating languages, summarizing text, answering questions, and analyzing feelings in reviews.
Why it mattersLLMs are the pinnacle of NLP. Knowing NLP basics helps you understand what problems LLMs solve.
Real exampleGoogle Translate — NLP analyzes grammar, context, and meaning to translate between 130+ languages.
🔨 Mini ProjectUse the free Hugging Face sentiment analysis demo to analyze the mood of 20 tweets or product reviews.
ResourcesHugging Face NLP Course (100% free), Stanford CS224N lectures on YouTube
🏔️ Large Language Models
1 week
Simply: LLMs are very large AI models trained on massive amounts of text data (like most of the internet). They learn patterns in language so well they can write essays, answer questions, code, translate, and reason — all from one model.
Why it mattersThis is the core topic of the entire roadmap. Everything else either explains how LLMs work or how to use them.
Real exampleChatGPT is built on GPT-4, an LLM. Claude, Gemini, and Llama are also LLMs.
🔨 Mini ProjectCompare 3 LLMs on the same task: ChatGPT, Claude, and Gemini. Ask them "Explain black holes to a 10-year-old." Note differences.
ResourcesAndrej Karpathy "Intro to LLMs" (YouTube, 1hr), Jay Alammar's blog
💡 How ChatGPT Works
3 days
Simply: ChatGPT was trained on text from books, Wikipedia, code, and websites. It learned to predict the next word, over and over, billions of times. Then it was fine-tuned using human feedback to be helpful, harmless, and honest. Every response is a very smart next-word prediction.
Why it mattersDemystifies AI. When you understand the process, you stop being scared and start seeing opportunities.
Real exampleType "The sun is a" into ChatGPT. It predicts "star" because that pattern appeared millions of times in training data.
🔨 Mini ProjectRead OpenAI's "How ChatGPT works" blog post and explain it in your own words (under 200 words) to someone.
ResourcesOpenAI official blog, "GPT in 60 lines of numpy" (GitHub)
🎫 Tokens Explained
2 days
Simply: LLMs don't read word by word. They read in "tokens" — chunks of characters. "ChatGPT" is 3 tokens: "Chat", "G", "PT". A token is roughly ¾ of a word. The model only sees tokens, never raw text. This affects cost and context limits.
Why it mattersEvery API call is charged by tokens. Understanding tokens saves money and helps you write better prompts.
Real exampleGPT-4 has a 128K token context window — roughly 96,000 words, or a short novel it can "remember" in one conversation.
🔨 Mini ProjectGo to platform.openai.com/tokenizer. Paste different texts and see how they tokenize. Notice how code tokenizes differently than English.
ResourcesOpenAI Tokenizer tool, "What are tokens?" by Andrej Karpathy
🏋️ Training vs Fine-tuning
3 days
Simply: Training = building a model from scratch on billions of texts. This costs millions of dollars and takes months. Fine-tuning = taking an already-trained model and teaching it your specific task with a smaller dataset. It's like hiring an expert and giving them company-specific training for 2 weeks.
Why it mattersYou'll fine-tune models for your projects. You'll never train from scratch. Knowing the difference matters for career conversations.
Real exampleA hospital fine-tunes an LLM on medical records so it understands doctor notes and drug names precisely.
🔨 Mini ProjectRead the OpenAI fine-tuning guide. Write a plan for fine-tuning a model for a customer support bot for an imaginary company.
ResourcesOpenAI fine-tuning docs, Hugging Face "Fine-tune a pretrained model" tutorial
✍️ Prompt Engineering Basics
1 week
Simply: Prompt engineering is the art of writing good instructions to get great outputs from AI. It's like learning how to ask questions clearly. A bad prompt gets a generic answer. A great prompt gets exactly what you need.
Why it mattersThis is the fastest skill to monetize. You can work as a prompt engineer without knowing how to code.
Real exampleBad: "Write an email." Better: "You are a startup founder. Write a 3-sentence cold email to a VC firm asking for a 20-min meeting about our AI health app."
🔨 Mini ProjectTake any topic you know well. Write 5 different prompts about it, from bad to excellent. Compare the outputs from ChatGPT.
ResourcesAnthropic's Prompt Engineering Guide, Learn Prompting (free website), OpenAI Cookbook
🐍 Python Basics for AI
2–3 weeks
Simply: Python is the programming language used for 95% of AI work. It's beginner-friendly with easy-to-read code. For AI, you need: variables, lists, dictionaries, loops, functions, and how to install/use libraries with pip.
Why it mattersEvery AI project in this roadmap uses Python. Even if you're a prompt engineer, basic Python lets you automate and build more.
Real exampleimport openai — this one line gives you access to GPT-4 in your Python code.
🔨 Mini ProjectWrite a Python script that calls the OpenAI API and has a simple back-and-forth conversation in the terminal. (10–20 lines of code).
ResourcesCS50P (free Harvard course), Automate the Boring Stuff with Python (free book), Python.org
📐 Math for AI (Basics)
1–2 weeks
Simply: You need three areas: (1) Linear Algebra — matrices and vectors are how data is stored. (2) Probability — AI makes predictions, not certainties. (3) Calculus — gradient descent, how models learn. You don't need to master all of this — just enough to read research papers and not feel lost.
Why it mattersYou can build AI apps without math. But math helps you understand what's happening, debug problems, and grow into senior roles.
Real exampleEvery embedding (like word2vec) is a vector — a list of numbers. Math helps you understand why similar words have similar vectors.
🔨 Mini ProjectWatch 3Blue1Brown "Essence of Linear Algebra" (YouTube, 15 short videos). Take notes with examples. No coding needed.
Resources3Blue1Brown (YouTube), Khan Academy (free), Mathematics for Machine Learning book (free PDF)
🔌 APIs Explained Simply
3–4 days
Simply: API stands for Application Programming Interface. Think of it as a waiter at a restaurant. You (your code) tell the waiter (API) what you want. The waiter goes to the kitchen (the server) and brings back the result. You don't need to know how the kitchen works — just how to order. OpenAI's API lets you use GPT-4 in your own apps with a few lines of code.
Why it mattersEvery AI project uses APIs. LangChain, OpenAI, Hugging Face, vector databases — they all communicate via APIs.
Real exampleWhen you use ChatGPT in your Python code: client.chat.completions.create(...) — you're calling OpenAI's API.
🔨 Mini ProjectGet a free OpenAI API key. Write a Python script that sends a message to GPT-4 and prints the response. 15 lines of code.
ResourcesOpenAI API docs (free), "APIs for Beginners" by freeCodeCamp (YouTube), Postman API learning center

Intermediate Level · Weeks 9–18
Core AI Engineering
Build real AI applications. Understand the architecture. Use professional tools.
🔄 Transformers Architecture
2 weeks
Simply: Transformers are the neural network architecture that powers ALL modern LLMs (GPT, Claude, Gemini, Llama). Introduced in the paper "Attention Is All You Need" in 2017, they process text in parallel (not word-by-word), making them fast and scalable to huge datasets.
Why it mattersEvery LLM you use is a Transformer. Understanding the architecture makes you a better engineer and differentiates you in interviews.
Real exampleGPT-4 is a Transformer with hundreds of billions of parameters (individual numbers the model learned during training).
🔨 Mini ProjectRead "The Illustrated Transformer" by Jay Alammar (free blog). Draw the architecture by hand. Then explain it to someone else.
ResourcesJay Alammar's blog, Andrej Karpathy "Let's build GPT" (YouTube), original paper (arxiv.org)
👁️ Attention Mechanism
1 week
Simply: Attention lets the model focus on the most relevant words in a sentence when understanding context. In "The bank of the river was muddy" vs "I went to the bank to get money" — attention helps the model know which "bank" meaning applies by looking at surrounding words.
Why it mattersSelf-attention is the core innovation of Transformers. It's what makes LLMs understand long-range context and relationships.
Real exampleWhen ChatGPT answers a complex question, attention keeps track of every relevant word across thousands of tokens.
🔨 Mini ProjectUse BertViz (open-source tool) to visualize how attention works on your own sentences. See which words attend to which.
Resources"Attention Is All You Need" paper, Jay Alammar "Visualizing Attention" blog
🧬 Embeddings
5 days
Simply: Embeddings convert words, sentences, or images into lists of numbers (vectors). Similar meanings get similar numbers. "King" and "Queen" have similar embeddings. "King - Man + Woman ≈ Queen" actually works mathematically. This is how AI understands meaning.
Why it mattersEmbeddings are the foundation of semantic search, RAG, recommendation systems, and vector databases.
Real exampleSpotify's recommendation engine converts songs into embeddings. Songs with similar embeddings get recommended together.
🔨 Mini ProjectUse OpenAI's text-embedding-3-small to embed 20 sentences. Compute similarity scores. Find which sentences are most similar.
ResourcesOpenAI embeddings guide, "Word2Vec explained" by Towards Data Science, Hugging Face MTEB benchmark
🗄️ Vector Databases
5 days
Simply: Vector databases store embeddings and let you search by meaning (not just keywords). You store thousands of documents as vectors. When the user asks a question, you convert the question to a vector and find the most similar documents. It's like a semantic Google search for your data.
Why it mattersEvery RAG system needs a vector database. Pinecone, ChromaDB, and Weaviate are used in almost every production AI app.
Real exampleA legal firm stores all their contracts as vectors. When a lawyer asks "find contracts with arbitration clauses," the vector search finds them by meaning.
🔨 Mini ProjectSet up ChromaDB (free, local). Load 50 Wikipedia paragraphs. Ask a question and retrieve the top 3 most relevant paragraphs.
ResourcesChromaDB docs (free), Pinecone learning center, Weaviate Academy (free)
🔍 RAG — Retrieval-Augmented Generation
2 weeks
Simply: RAG fixes a big problem with LLMs — they only know what they were trained on. RAG lets you connect the LLM to your own documents. Here's how: (1) User asks a question → (2) System searches your document database for relevant info → (3) Relevant info is added to the prompt → (4) LLM answers using your data + its training. It's like giving the LLM a cheat sheet before every question.
Why it mattersRAG is the #1 technique used in production AI apps. Almost every enterprise AI system uses it to connect LLMs to private data.
Real exampleA company builds a chatbot that answers employee questions using their internal HR policy documents — all via RAG, without fine-tuning.
🔨 Mini ProjectBuild a RAG system that can answer questions about a 50-page PDF. Use LangChain + ChromaDB + OpenAI. This is also your portfolio project.
ResourcesLangChain RAG tutorial, DeepLearning.AI "Building RAG" course (free), LlamaIndex docs
🔑 OpenAI API
1 week
Simply: The OpenAI API lets you use GPT-4, DALL-E, Whisper, and embeddings in your own apps. You send a request, pay per token, and get a response. It's the most widely-used AI API in the world, and knowing it well is essential for any AI developer.
Why it mattersIt's the industry standard. Most startups build their first AI features on OpenAI. Job postings often list it as a requirement.
Real exampleNotion's AI features, GitHub Copilot (originally), and thousands of startups use the OpenAI API.
🔨 Mini ProjectBuild a CLI tool that uses GPT-4 with system prompts, conversation history, and streaming responses. Use function calling to add a calculator tool.
ResourcesOpenAI Cookbook (GitHub), OpenAI official docs, OpenAI Developer Forum
🤗 Hugging Face
1 week
Simply: Hugging Face is the GitHub of AI models. It hosts 500,000+ free open-source models. You can download and run models locally with just a few lines of Python using their transformers library. It's the most important platform for AI developers after OpenAI.
Why it mattersFree alternatives to OpenAI, fine-tuning, datasets, model evaluation — Hugging Face is central to the entire open-source AI ecosystem.
Real examplefrom transformers import pipeline; classifier = pipeline("sentiment-analysis") — 2 lines to run an AI sentiment model.
🔨 Mini ProjectDownload 3 different models from Hugging Face Hub and compare them: one for text generation, one for summarization, one for translation.
ResourcesHugging Face Course (100% free), huggingface.co/docs, ML Engineering newsletter
🔗 LangChain
1–2 weeks
Simply: LangChain is a Python framework that makes it easy to chain together LLM calls, tools, and memory to build complex AI apps. Instead of writing hundreds of lines from scratch, LangChain gives you ready-made building blocks for chatbots, RAG systems, and AI agents.
Why it mattersThe most popular AI development framework. Listed in almost every AI job description. Speeds up development 10x.
Real exampleBuild a chatbot that: reads PDFs → stores in vector DB → answers questions with memory of past messages. All with LangChain in ~50 lines.
🔨 Mini ProjectRebuild your RAG project using LangChain. Then add conversation memory so the chatbot remembers previous questions in the same session.
ResourcesLangChain official docs, LangChain YouTube tutorials, Python LangChain by Greg Kamradt
⚙️ Fine-tuning Models
1–2 weeks
Simply: Fine-tuning adapts a pre-trained model to your specific task. You provide 50–1000 example input/output pairs, train the model on them, and it learns your style, domain, or behavior. Much cheaper than training from scratch. Results in a model that's specialized for your exact use case.
Why it mattersFine-tuned models outperform general models on specific tasks. This skill commands premium salaries in enterprise AI.
Real exampleA law firm fine-tunes GPT-4o-mini to draft contracts in their specific format and terminology, saving lawyers hours per week.
🔨 Mini ProjectFine-tune GPT-4o-mini using OpenAI's API on a dataset of 100 Q&A pairs about a topic you know well. Test and compare with the base model.
ResourcesOpenAI fine-tuning docs, Hugging Face "Fine-tune LLM" tutorial, Axolotl (open source fine-tuning tool)
🤖 AI Agents
1–2 weeks
Simply: AI Agents are LLMs that can take actions. Instead of just generating text, an agent can: search the web, write and run code, send emails, book meetings, click on websites. The LLM decides what tool to use and when, making multi-step decisions to complete a goal.
Why it mattersAgents are the next frontier of AI. They're moving from chatbots that talk to systems that act. This is where the field is heading.
Real exampleDevin (AI software engineer) can read a GitHub issue, write code to fix it, test it, and open a pull request — fully autonomously.
🔨 Mini ProjectBuild an agent using LangChain that can search the web, calculate math, and check the weather — based on what the user asks.
ResourcesLangGraph docs, AutoGPT GitHub, CrewAI docs, "Building AI Agents" by DeepLearning.AI (free)
🧠 Memory in AI Systems
4 days
Simply: LLMs have no memory between conversations by default — each chat starts fresh. To add memory, you store past messages and pass them back in each request. Types: (1) In-context — everything in the prompt. (2) External — stored in a database, retrieved when needed. (3) Summary — condense old messages to save tokens.
Why it mattersPersonalized AI assistants need memory. Any long-running agent or chatbot must handle memory efficiently.
Real exampleA customer support bot that remembers your past issues, account details, and previous complaints in every conversation.
🔨 Mini ProjectBuild a chatbot that stores conversation history in a SQLite database and loads past messages for context in new sessions.
ResourcesLangChain Memory docs, Mem0 (open-source memory for AI), MemGPT paper
🛠 Function Calling
4 days
Simply: Function calling lets you tell the LLM "here are tools you can use." The model decides when to call a tool, what arguments to pass, and uses the result to answer. Example tools: search_web(), get_weather(city), send_email(to, subject, body). The LLM acts like a smart orchestrator.
Why it mattersFunction calling is how AI agents interact with the real world. It's the bridge between LLMs and actual software systems.
Real exampleAsk ChatGPT "What's the weather in Mumbai?" — it calls a weather function internally and formats the result as a natural response.
🔨 Mini ProjectBuild a GPT-4 chatbot with 3 custom functions: get_stock_price(ticker), calculate(expression), search_wikipedia(query). Test it with complex questions.
ResourcesOpenAI function calling docs, LangChain tools guide, "Structured outputs" blog by OpenAI
📏 Model Evaluation
5 days
Simply: How do you know if your AI is actually good? Evaluation measures accuracy, fluency, factuality, and task performance. Techniques: human evaluation, automated metrics (BLEU, ROUGE, BERTScore), LLM-as-judge (use GPT-4 to grade outputs), and benchmark suites like MMLU and HellaSwag.
Why it mattersIn production, "vibes" aren't enough. You need quantitative metrics to improve systems and catch regressions after updates.
Real exampleAfter fine-tuning your customer service bot, you run 500 test cases and compare accuracy, response time, and user satisfaction scores.
🔨 Mini ProjectBuild an evaluation script that grades 50 AI responses on a 1–5 scale using GPT-4 as judge. Generate a report with average scores by category.
ResourcesEleutherAI LM Evaluation Harness, Ragas (RAG evaluation), DeepEval library
📦 Context Windows
3 days
Simply: The context window is how much text the LLM can "see" at once. It's like the model's working memory. GPT-4 Turbo = 128K tokens (~96K words). Claude 3.5 = 200K tokens. If your conversation + documents exceed this, old content gets dropped. Larger windows = more expensive.
Why it mattersContext limits affect your architecture decisions. Long documents, long conversations, and large codebases all need careful context management.
Real exampleFeeding a 300-page book into GPT-4 requires chunking it because the book exceeds the context window. You need RAG or summarization.
🔨 Mini ProjectWrite code to automatically chunk a long document, track token counts with tiktoken, and warn when you're near the context limit.
Resourcestiktoken library, "Lost in the middle" paper (research on long context), LangChain splitters docs
🛡️ AI Safety Basics
4 days
Simply: AI safety is about making AI systems that are helpful, harmless, and honest. Key concepts: hallucination (AI making up facts), bias (training data biases in outputs), prompt injection (malicious inputs trying to hijack the model), jailbreaking (bypassing safety rules), and alignment (making AI do what humans actually want).
Why it mattersEvery production AI system needs safety guardrails. Employers expect AI engineers to think about safety and misuse cases.
Real exampleA medical AI that confidently states wrong drug dosages (hallucination) could harm patients. Safety testing prevents deployment of such errors.
🔨 Mini ProjectRed-team your own chatbot. Try 20 adversarial prompts. Document which safety issues you find and how you'd fix them with system prompts or filters.
ResourcesAnthropic AI Safety blog, "AI Safety" by 80,000 Hours, OWASP LLM Top 10 (free)

Production AI Systems
Build, optimize, and deploy real-world AI systems at scale.
🏗️ Build Your Own LLM App
2 weeks
Simply: This is where everything comes together. You design an end-to-end AI application: backend (FastAPI), LLM integration (OpenAI/LangChain), data storage (vector DB + SQL), frontend (Next.js or Streamlit), authentication, and deployment. A production LLM app is a system, not just a chatbot.
🔨 ProjectBuild a "Smart Research Assistant" — takes any URL, extracts content, stores in vector DB, answers questions, cites sources, exports PDF report. Deploy on Railway or Vercel.
ResourcesFull Stack LLM Bootcamp (free, YouTube), FastAPI docs, Streamlit docs
🕸️ Multi-Agent Systems
2 weeks
Simply: Instead of one AI agent doing everything, you create multiple specialized agents that collaborate. A "team" of AIs: a Researcher agent finds information, a Writer agent writes, an Editor agent reviews, a Publisher agent formats and posts. Each agent is an LLM with a specific role, tools, and instructions.
🔨 ProjectBuild a 3-agent content pipeline using CrewAI or LangGraph: Researcher → Writer → SEO Editor. Input a topic, output a fully-written, optimized blog post.
ResourcesCrewAI docs (free), LangGraph tutorials, "Multi-agent systems" by DeepLearning.AI
🗜️ Quantization
1 week
Simply: LLMs store numbers with 32-bit precision. Quantization reduces this to 8-bit or 4-bit, making models 4-8x smaller and faster — with minimal quality loss. A 70B model that needs 4 expensive GPUs can run on 1 consumer GPU after 4-bit quantization. Essential for running models locally.
🔨 ProjectUse bitsandbytes to load a 7B Hugging Face model in 4-bit and compare: memory usage, speed, and output quality vs full-precision version.
Resourcesbitsandbytes library, "Quantization explained" by Hugging Face, GGUF format docs
🎯 LoRA & PEFT
1–2 weeks
Simply: LoRA (Low-Rank Adaptation) is a smarter way to fine-tune large models. Instead of updating all billions of parameters (expensive), LoRA adds small "adapter" layers and trains only those — using 100x less memory. PEFT (Parameter-Efficient Fine-Tuning) is the umbrella term for techniques like LoRA, QLoRA, and adapters.
🔨 ProjectFine-tune Llama 3.1 8B using QLoRA on Google Colab (free GPU). Use a custom dataset of 500 examples. Compare with base model. Total cost: $0.
ResourcesHugging Face PEFT library, "LoRA Explained" paper, Axolotl fine-tuning framework
🎓 RLHF — Explained Simply
1 week
Simply: RLHF (Reinforcement Learning from Human Feedback) is how ChatGPT was made helpful and safe after initial training. Human raters rank AI responses from best to worst. A "reward model" learns from these rankings. Then the LLM is trained to maximize reward using reinforcement learning. This is why ChatGPT is polite, refuses harmful requests, and tries to be helpful.
🔨 ProjectImplement a simplified RLHF loop: collect 100 human preferences on AI response pairs, train a reward model, use it to rank new responses.
ResourcesInstructGPT paper, "RLHF explained" by Hugging Face, TRL (Transformer RL) library
⚡ Model Optimization
1 week
Simply: Making models faster and cheaper without losing quality. Techniques: (1) Caching — store repeated computations. (2) Batching — process multiple requests together. (3) Speculative decoding — use a small model to predict tokens, big model verifies. (4) KV cache — remember attention computations. (5) Flash Attention — faster attention algorithm.
🔨 ProjectBenchmark your LLM app: measure latency (time per response) and throughput (requests per second). Apply caching and batching. Measure improvement.
ResourcesvLLM docs, Flash Attention GitHub, "LLM Inference" by Hugging Face blog
🎮 GPU Basics for AI
5 days
Simply: GPUs are specially designed chips for AI math. A CPU has 16 powerful cores. A GPU has 10,000 simple cores that do math in parallel — perfect for matrix operations in neural networks. NVIDIA GPUs (A100, H100, RTX 4090) are the industry standard. Cloud providers rent them by the hour: AWS, GCP, Lambda Labs.
🔨 ProjectRent a GPU on Lambda Labs ($0.50/hr). Run a model inference benchmark, compare throughput on GPU vs CPU. Practice CUDA basics with a simple matrix multiply.
ResourcesNVIDIA Deep Learning Institute (free courses), "GPU explained" by Linus Tech Tips, PyTorch CUDA docs
🔄 Inference Pipelines
1 week
Simply: An inference pipeline is the complete system that takes a user's request, processes it, runs it through the model, and returns the result — reliably, at scale. It includes: load balancer → API gateway → model server → output formatter → monitoring. Tools: vLLM, TGI (Text Generation Inference), Triton Inference Server.
🔨 ProjectDeploy a 7B model using Hugging Face Text Generation Inference (TGI). Add a simple FastAPI wrapper. Load test with 50 concurrent users.
ResourcesvLLM docs (open source), Hugging Face TGI docs, Triton Inference Server docs
☁️ Deploying AI Apps
1–2 weeks
Simply: Taking your AI app from your laptop to the internet so others can use it. Steps: (1) Containerize with Docker, (2) Choose cloud platform (AWS, GCP, Railway, Fly.io), (3) Set up environment variables for API keys, (4) Add monitoring and logging, (5) Handle scaling for traffic spikes. Start simple with Railway or Fly.io, move to AWS for production.
🔨 ProjectDockerize your RAG chatbot. Deploy it on Railway with a Streamlit frontend. Add Sentry for error tracking. Share the public URL in your portfolio.
ResourcesDocker official docs, Railway.app (easiest), "Deploying LLM apps" by Towards Data Science
📈 Scaling AI Systems
1 week
Simply: What happens when 1,000 users use your AI app at the same time? Scaling handles this. Key concepts: horizontal scaling (add more servers), load balancing (spread requests), queuing (handle bursts), caching (avoid redundant LLM calls), async processing (don't block the user), and cost management (LLM APIs get expensive at scale).
🔨 ProjectAdd a Redis queue to your AI app. Implement request deduplication (cache identical questions for 1 hour). Track cost-per-request. Simulate 100 concurrent users.
ResourcesRedis docs, Celery (task queue), "Scaling AI" case studies by Anthropic and OpenAI blogs

🌱 Beginner Questions

Q1What is the difference between AI, ML, and Deep Learning?
AI is the broad field of making computers intelligent. ML is a subset of AI where computers learn from data without explicit programming. Deep Learning is a subset of ML that uses multi-layered neural networks. Think of it as nested circles: AI ⊃ ML ⊃ Deep Learning. LLMs are a specific type of Deep Learning model.
Q2What is a token? How many tokens is roughly 1 page of text?
A token is a chunk of text that the model processes — roughly ¾ of a word. One page of text (approximately 250 words) is about 330–400 tokens. The rule of thumb: 1 token ≈ 4 characters in English. Code, URLs, and non-English text can tokenize differently. You pay for input + output tokens when using the OpenAI API.
Q3What is the difference between training and fine-tuning an LLM?
Training = building a model from scratch on hundreds of billions of tokens. Requires thousands of GPUs, months of time, and millions of dollars. Fine-tuning = taking an already-trained model and updating it on a smaller, task-specific dataset (50–10,000 examples). You train only the last few layers or use efficient methods like LoRA. Accessible to any developer with ~$50 and Google Colab.

⚡ Intermediate Questions

Q4Explain RAG and when you would use it vs. fine-tuning.
RAG (Retrieval-Augmented Generation) retrieves relevant documents from a knowledge base at query time and adds them to the prompt. Use RAG when: data changes frequently, you need source citations, data is private/large, or you need to update knowledge without retraining. Use fine-tuning when: you need a specific style/format, consistent behavior on a narrow task, faster inference, or the model needs to "bake in" specialized knowledge. Many production systems use both.
Q5What is self-attention and why does it matter?
Self-attention is the mechanism in Transformers that lets every token in a sequence attend to (look at) every other token when computing its representation. For each token, the model computes Query, Key, and Value vectors. Attention scores = dot product of Query with all Keys, softmaxed. This lets the model understand "bank" differently in "river bank" vs "savings bank" by looking at surrounding context. It's parallelizable (unlike RNNs), which is why Transformers scale so well.
Q6What are embeddings and how are they used in LLM applications?
Embeddings are dense numerical vectors that represent the semantic meaning of text. Similar texts have similar (close) vectors in embedding space. In LLM apps, embeddings enable: (1) Semantic search — find documents by meaning, not keywords. (2) RAG — retrieve relevant context from a knowledge base. (3) Clustering — group similar documents. (4) Recommendation — find similar items. They're created using embedding models like OpenAI's text-embedding-3-small and stored in vector databases like Pinecone or ChromaDB.

🚀 Advanced Questions

Q7Explain LoRA and why it's preferred over full fine-tuning.
LoRA (Low-Rank Adaptation) freezes the original pre-trained model weights and injects trainable low-rank decomposition matrices into each Transformer layer. Instead of updating all parameters (e.g., 7 billion in a 7B model), LoRA trains ~0.1-1% of parameters in small adapter matrices. Benefits: 100x less GPU memory, 3-4x faster training, multiple LoRA adapters per base model (like plugins), and comparable performance to full fine-tuning on most tasks. QLoRA combines LoRA with 4-bit quantization for even lower memory usage.
Q8What is hallucination in LLMs and how do you mitigate it?
Hallucination is when an LLM generates confident-sounding but factually incorrect information. It happens because LLMs are trained to produce plausible text, not necessarily true text. Mitigation strategies: (1) RAG — ground answers in retrieved documents. (2) Self-consistency — generate multiple answers and take the majority. (3) Verification chains — use a second LLM call to fact-check. (4) Constrained generation — force the model to only use information from provided context. (5) Temperature reduction — lower temperatures give more conservative outputs. (6) Prompt engineering — "Only answer based on the provided context. Say 'I don't know' if unsure."
Q9How would you design a production AI system that handles 10,000 requests per day?
Key architecture decisions: (1) API Gateway — rate limiting, authentication, routing. (2) Queue — use Redis/Celery for async processing, handle traffic bursts. (3) Caching — cache identical or similar queries (Redis with TTL). (4) Load balancing — distribute across multiple model instances. (5) Monitoring — track latency, error rates, token costs per endpoint with Prometheus/Grafana. (6) Cost optimization — use GPT-4o-mini for simple tasks, GPT-4 only for complex ones. (7) Fallbacks — if primary model fails, fall back to backup. At 10K requests/day (~7/minute average), a single API instance handles this easily. Plan for 10x spikes.

// Key Terminology — The LLM Glossary
LLMLarge Language Model — AI trained on massive text to understand and generate language. Examples: GPT-4, Claude, Llama, Gemini.
TokenThe unit of text LLMs process. ~¾ of an English word. 1 page ≈ 400 tokens. Cost and context limits are measured in tokens.
Context WindowMaximum tokens the model can process at once. GPT-4: 128K. Claude 3.5: 200K. Think of it as working memory.
EmbeddingA vector (list of numbers) representing text meaning. Similar texts have similar embeddings. Foundation of semantic search.
RAGRetrieval-Augmented Generation. Retrieve relevant documents → add to prompt → LLM answers with your data.
Fine-tuningFurther train a pre-trained model on custom data. Adapts style, format, or domain knowledge.
LoRALow-Rank Adaptation. Efficient fine-tuning that updates <1% of parameters. Makes fine-tuning accessible on consumer hardware.
RLHFReinforcement Learning from Human Feedback. How ChatGPT was made helpful and safe using human preference data.
HallucinationWhen LLM generates confident but factually wrong information. Mitigate with RAG, verification, and constrained prompts.
TemperatureControls randomness. 0 = deterministic/focused. 1 = creative/random. 0.7 is common for creative tasks.
Prompt InjectionMalicious input tries to override system instructions. "Ignore previous instructions and..." — a key security concern.
QuantizationReduce model precision (32-bit → 4-bit) to run larger models on less GPU memory with minimal quality loss.
AgentLLM that can take actions using tools (web search, code execution, APIs). Makes multi-step decisions autonomously.
Vector DatabaseDatabase optimized for storing and searching vectors by similarity. Examples: Pinecone, ChromaDB, Weaviate, Qdrant.
// Prompt Engineering Cheat Sheet
Role + Task"You are a senior Python developer. Review this code for bugs and performance issues: [code]"
Few-shotGive 2-3 examples of input/output before your actual request. The model follows the pattern.
Chain of ThoughtAdd "Let's think step by step." at the end. Dramatically improves reasoning on math and logic problems.
Output Format"Respond in JSON with fields: name, age, summary. No other text." Constraints improve consistency.
Constraints"In exactly 3 bullet points." "Using only simple words a 10-year-old would understand." "Under 100 words."
Negative Examples"Do NOT use jargon. Do NOT use bullet points. Do NOT repeat information already given."

❌ Mistake 1Skipping the foundations. If you don't understand what a neural network is, advanced topics will confuse you.
❌ Mistake 2Only watching tutorials, never coding. You must build things. Videos are 20% of learning. Coding is 80%.
❌ Mistake 3Trying to build your own LLM from scratch. Don't. Use existing models via APIs. Build applications, not models.
❌ Mistake 4Not using GitHub from day 1. Every project you build should be committed and public. This is your portfolio.
